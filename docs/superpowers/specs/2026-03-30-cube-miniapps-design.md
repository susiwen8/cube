# 原生微信/支付宝魔方小程序设计

## 目标

构建一个同时支持微信小程序和支付宝小程序的 3x3 魔方应用，提供：

- 单指拖动色块转动对应层
- 单指拖动画布空白区域转动视角
- 打乱、复位、自动求解
- 教学模式
- 计时模式和本地成绩记录

项目以共享核心引擎为主，微信与支付宝使用各自原生小程序壳接入，保证逻辑一致和后续维护成本可控。

## 用户体验

用户进入应用后会直接看到一个可操作的 3D 魔方视图。主页面不切换魔方实例，只在底部切换不同模式面板：

- 自由玩：直接操作魔方，支持打乱、复位、求解
- 教学：展示分阶段的入门说明、公式、当前目标和高亮层
- 计时：点击打乱并开始，第一次有效转动后开表，复原后停表
- 自动求解：逐步回放解法，支持自动播放、暂停、单步前进

## 交互设计

### 手势

- 触摸命中某个可见贴纸时，进入候选转层态
- 拖动超过阈值后，根据命中面的局部坐标和拖动方向映射到一个标准转动
- 松手时根据当前拖动角度决定回弹还是吸附到 90 度
- 触摸未命中贴纸时，该次拖动解释为视角旋转
- 视角使用受限轨道相机，限制俯仰角，避免用户翻转到难以理解的角度

### 动画

- 层转动采用逐帧插值，保证拖动时跟手
- 自动求解和教学演示复用同一套动画队列
- 当前教学目标层使用高亮，非目标层降低亮度

## 架构

### shared/core

共享核心只使用原生 JavaScript 模块，不依赖第三方库，供 Node 测试和双端小程序共同使用。

- `cube-state.js`：3x3 魔方状态模型，维护每个小块的位置和朝向
- `moves.js`：标准转动定义、转动应用、逆序、压缩、历史回放
- `scramble.js`：随机打乱序列生成
- `solver.js`：基于规范化历史的自动复原步骤生成
- `tutorial.js`：教学章节、公式、阶段说明和高亮配置
- `timer.js`：计时状态机、步数统计、成绩记录
- `session.js`：聚合当前模式、求解队列、动画状态和教学进度

### shared/render

- `math.js`：向量、矩阵、旋转计算
- `projection.js`：3D 到 2D 的透视投影
- `hit-test.js`：屏幕触点到可见贴纸的命中检测
- `cube-renderer.js`：绘制当前状态、动画层、教学高亮
- `gesture-mapper.js`：贴纸触点和拖动向量到标准转动的映射

### miniapps/wechat 与 miniapps/alipay

各端只保留薄适配层：

- 页面生命周期
- Canvas 上下文初始化
- 触摸事件格式标准化
- 本地存储 API 适配
- 控件面板与模式切换

## 自动求解

第一版自动求解使用“小程序内合法操作历史反演”策略：

- 魔方从初始解状态开始
- 所有打乱和手势操作都记录为标准转动
- 当前状态的自动解法直接由规范化历史取逆得到
- 求解步骤经过合并和规范化后用于动画播放

这样可以保证：

- 只要状态由应用内合法转动产生，就一定可还原
- 求解逻辑稳定、易验证、易教学演示
- 不需要引入体积更大的通用搜索求解器

## 教学模式

教学模式不依赖当前自动求解结果，而是提供稳定的入门教程：

1. 认识六个面和转动记号
2. 白十字目标
3. 白角归位
4. 中层棱块
5. 顶层十字
6. 顶层方向调整
7. 顶层换位与完成

每一课包含：

- 当前目标
- 操作解释
- 公式文本
- 重点高亮层
- 可选演示动作

## 计时与成绩

- `打乱并开始` 进入待开表态
- 第一次有效层转动时正式计时
- 每次合法转动计入步数
- 复原后自动停表
- 使用自动求解或教学辅助时，成绩标记为辅助完成，不写入最佳成绩
- 成绩只保存到本地

## 页面结构

- 顶部：模式标签、当前计时、步数、当前状态说明
- 中部：Canvas 魔方视图区
- 底部：操作按钮和模式面板
- 辅助面板：教学说明、求解步骤、历史成绩

## 测试策略

优先覆盖共享核心和渲染映射逻辑：

- 转动后每个小块位置和朝向正确
- 转动逆序可恢复原状态
- 打乱结果合法且不出现连续同轴无意义动作
- 自动求解能从应用内任意历史状态复原
- 教学章节配置完整，阶段和提示文本可用
- 计时状态机在开始、暂停、完成、辅助完成场景下行为正确
- 手势映射在关键贴纸和拖动方向组合下输出正确标准动作

## 非目标

- 首版不接云端排行榜
- 首版不做通用最优求解器
- 首版不做多魔方阶数切换
- 首版不引入 WebGL 或第三方 3D 引擎
